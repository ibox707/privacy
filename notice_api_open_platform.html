2023-09-07

阿里云盘开放平台立场与合规使用（阿里云盘 API）

近期收到用户反馈账号因「涉嫌违规」被「限制登录」，阿里云盘开放平台立场与原则如下（适用于调用阿里云盘 API 的开发者）。

我们希望

  • 为开发者提供开发友好的 阿里云盘 OpenAPI，帮助以更低成本集成阿里云盘个人云存储，与开发者共同成长；
  • 为用户提供长期、稳定、好用的云存储服务，保障数据安全与个人隐私。

我们禁止

  • 禁止分享开发者账号（含 appid 与 secret）；
  • 禁止分享个人账号（含用户 token）；
  • 禁止分享文件下载地址与视频播放地址；
  • 禁止不规范接入导致泄露用户隐私的行为。
请自查并及时停止：图床/文件分发、将播放链接挂到外网做视频网站、将开发者账号共享给终端用户、替代 OSS 做企业服务等。安全团队会实时监控，违规可能封号且无法解封。
2023-08-03

「备份盘」与「资源库」空间拆分（V4.9.1）与阿里云盘 API

阿里云盘客户端升级后，原「文件」拆分为两个空间：「备份盘」与「资源库」。以下与 阿里云盘 API 中「用户信息」「文件列表」「播放」等能力相关。

Q：为什么有两个存储空间？区别是什么？

备份盘：独立空间，存自己备份与上传的文件；上传/下载/列表/预览与原「文件」一致；不支持公开分享，支持快传；不支持复制到「资源库」；手机/电脑/微信/钉钉备份文件会上传至此。

资源库：从公开分享转存的文件默认存此处；上传/下载/列表/预览一致；支持公开分享与快传；支持复制、移动到「备份盘」。（LV1 以上分享达人升级后默认存「资源库」，default_drive_id 默认也为资源库。）

Q：对「获取用户文件列表与详情」的影响？

可通过阿里云盘 API「用户信息」相关接口获取「备份盘」与「资源库」的 drive_id。若不做处理，云盘侧兼容：default_drive_id 等效于「备份盘」的 drive_id。若用户登录过新版本，可拿到两个 drive_id；未登录过新版本则仅有 default_drive_id。

Q：对「获取用户最近播放记录」的影响？

视频播放相关阿里云盘 API：返回结果为「备份盘」与「资源库」播放记录的并集（8.4 上线说明）；此前可能仅展示 default_drive_id 侧记录，以官方文档为准。

Q：复制与移动限制？

备份盘 → 资源库：不支持复制与移动（阿里云盘 API 后续会做限制）。资源库 → 备份盘：支持复制与移动；文件被冻结则移动失败。

Q：三方应用用户侧影响？

若转存文件在应用中看不到，可能是应用默认读「备份盘」；可将文件从「资源库」移到「备份盘」。

2023-06-30

阿里云盘 API：下载链接过期时间参数调整

业务升级，计划对阿里云盘 OpenAPI 路径 /adrive/v1.0/openFile/getDownloadUrl（获取下载链接等）的下载过期时间参数做调整（详见阿里云盘开放平台「获取文件详情」等文档）。

  • 原：最长 115,200 秒（32 小时），默认 900 秒；
  • 调整后：最长 1800 秒（0.5 小时），默认仍为 900 秒；
  • 传入 < 1800 秒：本次调整无影响；
  • 传入 > 1800 秒：将按 1800 秒处理；
  • 该阿里云盘 API 调整时间：8 月 14 日（以公告当年为准）。
建议：下载地址更新逻辑按 x-oss-expires 设置过期时间；若此前已按此处理则无影响；若写死过期时间，请尽快调整调用方式。

通用说明：阿里云盘 API 限流与密钥安全

  • 阿里云盘 API 可能按频率、并发等维度限流；超限可能返回 403 / 429 等，请以阿里云盘开放平台官方文档为准。
  • 请妥善保管阿里云盘开放平台颁发的 appid、secret 及用户 token，勿在客户端明文暴露或对外分享。
  • 阿里云盘 API 废弃与变更请关注开放平台公告与迁移说明。
